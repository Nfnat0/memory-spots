# Memory Photo Palace 次フェーズ実装計画書

前提: MVPは実装済み。  
本フェーズでは復習機能の強化は後回しにし、触って楽しい体験を優先する。

## 1. 目的

アプリを「資格勉強向けの無機質な暗記ツール」ではなく、  
自分の写真と地図を使って記憶空間を作る楽しいアプリにする。

重点は以下。

1. 地図表示を最優先で実装する
2. 写真・場所・メモを探索する体験を良くする
3. 写真上の視覚メモ表現を強化する
4. 復習機能の強化は本フェーズでは対象外にする

---

## 2. 実装方針

本フェーズの主役は「地図」と「写真上の視覚メモ」。

ユーザーがアプリを開いたときに、  
自分の記憶スポットが地図上に並んでいて、触るだけで楽しい状態を目指す。

学習アプリ感を出しすぎない。  
UI文言も「復習」「正答率」「苦手」より、「スポット」「テーマ」「メモ」「アルバム」「記憶マップ」を優先する。

---

## 3. 優先実装

## 3.1 地図表示

### 目的

写真に紐づいた場所を地図上で見られるようにする。  
アプリの入口を「記憶マップ」として魅力的にする。

### 実装内容

- MapKitで地図画面を追加する
- 位置情報を持つMemoryPhotoを地図上にピン表示する
- ピンには写真サムネイルを表示する
- ピンをタップすると写真プレビューカードを表示する
- プレビューカードからPhotoEditorViewへ遷移する
- 位置情報がない写真は地図には表示しない
- 地図画面から記憶セット詳細へ移動できる

### UI要件

- 通常のピンではなく、写真サムネイル付きの丸いピンにする
- 選択中のピンは少し拡大する
- 下部に写真プレビューカードを表示する
- 地図上部に「記憶マップ」タイトルを表示する
- セットまたはテーマで絞り込みできる導線を用意する

### 完了条件

- 位置情報付き写真が地図上に表示される
- ピンから写真編集画面へ遷移できる
- 複数の記憶セットの写真を地図上で見られる
- セット単位で表示を絞り込める

---

## 3.2 写真追加時の位置情報体験改善

### 目的

写真と場所の紐づけを自然にする。

### 実装内容

- 写真ライブラリから選んだ画像に位置情報があればMemoryPhotoに保存する
- カメラ撮影時は現在地を取得してMemoryPhotoに保存する
- 位置情報がない場合は、任意で地図から場所を選べるようにする
- 位置情報なしでも写真登録は可能にする

### UI要件

- 位置情報あり: 「この場所に保存しました」と軽く表示する
- 位置情報なし: 「場所を追加する」ボタンを表示する
- 位置情報の取得失敗をエラー扱いにしすぎない

### 完了条件

- ライブラリ写真または撮影写真に位置情報を紐づけられる
- 位置情報なし写真も問題なく保存できる
- 後から場所を追加・変更できる

---

## 3.3 UI/UX改善

### 目的

実用性よりも、触りたくなる軽快さを優先する。

### 実装内容

- HomeViewの第一導線を「記憶マップ」にする
- セット一覧は補助導線として残す
- 写真カードのデザインを改善する
- テーマ切り替えUIをセグメントまたはチップ形式にする
- 空状態の画面を改善する
- 追加ボタンを分かりやすくする

### UI要件

- 全体的に柔らかいカードUIにする
- 写真を大きく見せる
- テキスト量を減らす
- 「学習」「復習」より「作る」「見る」「置く」「めぐる」系の言葉を使う
- 空状態では、何をすればよいかを短く案内する

### 画面構成の優先度

1. MapView
2. MemorySetDetailView
3. PhotoEditorView
4. HomeView
5. ReviewView

ReviewViewは本フェーズでは大きく触らない。

### 完了条件

- 起動後すぐに記憶マップへアクセスできる
- 写真・テーマ・メモの切り替えが直感的になる
- 空状態でも次にやることが分かる

---

## 3.4 画像・視覚メモ強化

### 目的

写真上に置ける要素を増やし、コラージュ的に楽しくする。  
ただし複雑な画像編集アプリにはしない。

### 実装内容

既存のテキストメモに加えて、以下の視覚メモを追加する。

- 色付き付箋
- 小さな画像メモ
- アイコンメモ
- 番号ラベル
- 矢印

### MemoryItemの拡張案

```swift
enum MemoryItemType: String, Codable {
    case stickyText
    case image
    case icon
    case numberLabel
    case arrow
}
```

追加プロパティ候補。

```swift
var type: String
var colorName: String?
var iconName: String?
var imagePath: String?
var rotation: Double
var scale: Double
```

### 操作

- 追加メニューから種類を選べる
- ドラッグ移動できる
- タップで編集できる
- 長押しで削除メニューを出す
- ピンチ拡大縮小は余裕があれば実装する
- 回転は余裕があれば実装する

### UI要件

- 追加メニューは楽しい見た目にする
- 「付箋」「画像」「アイコン」「番号」「矢印」を並べる
- 写真上の要素が多すぎる場合でもテーマ切り替えで整理できるようにする

### 完了条件

- テキスト付箋以外の視覚メモを追加できる
- 追加した視覚メモが保存・再表示される
- 既存のテーマ切り替えと矛盾しない

## 5. 推奨実装順

### Step 1: 地図画面の追加

- MapViewを作る
- MemoryPhotoの位置情報を読み込む
- サムネイルピンを表示する
- ピンタップでプレビューカードを表示する

### Step 2: 写真と位置情報の紐づけ改善

- ライブラリ写真の位置情報取得を実装する
- 撮影時の現在地保存を実装する
- 後から場所を追加・変更できる画面を作る

### Step 3: ホーム導線の変更

- 起動後の主導線を記憶マップ寄りにする
- セット一覧はタブまたは下部導線にする
- 空状態UIを改善する

### Step 4: 写真編集画面の体験改善

- テーマチップUI
- 付箋追加UI改善
- メモ選択時の見た目改善
- 写真の表示領域を広げる

### Step 5: 視覚メモの追加

- 色付き付箋
- 番号ラベル
- アイコン
- 画像メモ
- 矢印

---

## 6. 受け入れ条件

- アプリ内で地図を開き、位置情報付き写真を確認できる
- 地図上の写真ピンから写真編集画面に移動できる
- 写真追加時に位置情報を保存できる
- 位置情報がない写真にも後から場所を設定できる
- UIが学習ツール寄りではなく、写真・地図・記憶空間づくり寄りになっている
- 写真上に複数種類の視覚メモを置ける
- 既存の記憶セット、テーマ、メモ機能が壊れていない
