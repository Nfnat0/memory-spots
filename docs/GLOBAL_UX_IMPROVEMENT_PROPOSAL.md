# Memory Spots: Global UX Improvement Proposal

Last updated: 2026-06-05

## 1. ビジョン

Memory Spots は「写真の道を、記憶のノートにする」アプリです。現在の MVP は、資格勉強に閉じた体験から、地図・写真・視覚メモを中心にしたグローバル向けの空間記憶ツールへ移行済みです。

今後の UX 方針は、説明文を増やすことではなく、ユーザーが写真と場所を触っているうちに自然に理解できる体験を磨くことです。

## 2. 完了済みの改善

- アプリ名を Memory Spots に統一
- 日本固有の AWS/五反田シードから、My Room、Desk Setup、Local Park のサンプルへ変更
- 初回起動時のインタラクティブチュートリアルを追加
- HomeView を Memory Map と Albums のタブ構成に変更
- MapKit のサムネイルピンと写真プレビューカードを追加
- アルバム/テーマのフィルタを追加
- 写真の位置情報追加・変更 UI を追加
- テキスト以外の視覚メモを追加: 画像、アイコン、番号、矢印
- `Localizable.xcstrings` と `InfoPlist.xcstrings` を導入
- App Store 用のサポート/プライバシーページを GitHub Pages で公開できる構成に整理

## 3. 次に磨くべき UX

### 3.1 言葉の密度を下げる

英語圏でも日本語圏でも、画面上の言葉は短く保ちます。

推奨する用語:

- Memory Map
- Albums
- Themes
- Notes
- Waypoints
- Review Notes

避けたい方向:

- 成績管理アプリに見える文言
- 長すぎるチュートリアル文
- ボタン内の説明文

### 3.2 レビューを「採点」より「思い出す散歩」に寄せる

現在のレビューは、置いた視覚メモをタップして答えを表示する形です。この方向は Memory Spots らしいので維持します。

改善する場合も、まずは以下に留めます。

- 写真上の場所を思い出してから答えを見る導線
- 軽いハプティクス
- 必要なら簡易結果ボタン

高度な SRS や成績分析は、アプリの焦点が変わるため現時点では入れません。

### 3.3 位置情報なしの体験を気持ちよくする

写真に位置情報がないことはエラーではありません。地図に出ない理由を短く伝え、後から場所を置けることを常に分かるようにします。

改善候補:

- アルバム内で「場所なし」写真が見つけやすい表示
- 空の Memory Map からアルバムや場所編集に進む導線
- ライブラリ写真の位置情報がない場合の柔らかい説明

### 3.4 アクセシビリティを仕上げる

視覚メモが主役のアプリなので、VoiceOver と Dynamic Type の確認は重要です。

- メモタイプと表示テキストを VoiceOver で読めるか
- 写真サムネイルピンの名前が分かるか
- テーマチップが長文翻訳で崩れないか
- 地図や写真上の操作に代替導線があるか

## 4. App Store 向けの見せ方

スクリーンショットでは、機能説明よりも「場所にメモを置く楽しさ」を見せます。

おすすめ構成:

1. Memory Map
2. アルバム詳細とテーマチップ
3. 写真編集と視覚メモ
4. レビューで答えを表示
5. 初回チュートリアル

## 5. 総括

Memory Spots の魅力は、写真・場所・メモが一体になることです。次フェーズでは機能を増やすより、地図から写真へ、写真からメモへ、メモから記憶へ進む流れをより軽くします。
